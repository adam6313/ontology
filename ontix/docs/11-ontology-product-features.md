## 1. 產品定位躍升

### Before: KOL 數據庫 + 搜尋工具

```
用戶需求：「我想知道麥當勞在社群上的評價」

現有系統回應：
├── 搜尋「麥當勞」→ 找到 1,234 則貼文
├── 搜尋「McDonald's」→ 找到 567 則貼文（用戶要自己知道要搜這個）
├── 搜尋「麥當當」→ 找到 89 則貼文（用戶可能不知道這個暱稱）
└── 結果：用戶需要手動合併，無法得到完整圖像
```

### After: Brand Intelligence Platform

```
用戶需求：「我想知道麥當勞在社群上的評價」

Ontology 系統回應：
┌─────────────────────────────────────────────────────────────┐
│  Brand: 麥當勞 (McDonald's)                                  │
│  ─────────────────────────────────────────────────────────  │
│  別名：McDonald's, 麥當當, 麥麥, 金拱門                        │
│  總提及：1,890 則（系統自動合併所有別名）                       │
│                                                              │
│  情緒分布                     熱門 Aspects                    │
│  ██████████ 正面 67%         薯條 ████████ 89% 正面          │
│  ████ 中性 25%               服務 ████ 45% 正面              │
│  █ 負面 8%                   價格 ██████ 62% 負面            │
│                                                              │
│  提及此品牌的 Top KOL                                         │
│  1. @美食家小明 (32 則, 正面為主)                              │
│  2. @吃貨日記 (28 則, 中性為主)                                │
│  3. @台北覓食 (19 則, 正面為主)                                │
│                                                              │
│  趨勢：本週 +23% ↑  |  建議：聲量上升中，適合加碼行銷           │
└─────────────────────────────────────────────────────────────┘
```

### 定位對比

| 維度 | 現有 KOLr | Ontology KOLr |
|------|-----------|---------------|
| **產品定位** | KOL 數據庫 + 搜尋 | **Brand Intelligence Platform** |
| **核心價值** | 幫你找 KOL | 告訴你品牌在社群的完整故事 |
| **數據粒度** | 貼文層級 | **Entity 層級**（品牌/產品/KOL） |
| **查詢方式** | 關鍵字搜尋 | **語意查詢**（自動合併別名） |
| **分析深度** | 貼文數量統計 | **多維度洞察**（情緒/Aspect/趨勢/關係） |
| **競爭壁壘** | 數據量 | **Entity Graph + 語意理解** |

| 指標 | 現有系統 | Ontology 系統 |
|------|----------|---------------|
| 品牌提及數 | 需搜尋多個關鍵字手動加總 | **自動合併所有別名** |

| 趨勢分析 | 手動拉數據 | **自動追蹤，標記關鍵事件** |

#### 使用情境

**情境 1：品牌方找 KOL**
```
品牌方需求：「我是雅詩蘭黛，想找美妝 KOL 合作」

Ontology 系統回應：
├── @Lisa的日常：提過雅詩蘭黛 29 次，但未合作過 → 潛力合作對象
├── @美妝控小美：提過雅詩蘭黛 45 次，已合作 3 次 → 深度合作對象
└── @成分黨研究員：提過競品蘭蔻 67 次，未提雅詩蘭黛 → 可能是競品愛好者
```

**情境 2：KOL 競品分析**
```
品牌方需求：「我想知道這個 KOL 是否也推過競品」

Ontology 系統回應：
@Lisa的日常 品牌提及分析：
├── 你的品牌（SK-II）：47 次，情緒 4.8
├── 競品 A（蘭蔻）：38 次，情緒 4.5
├── 競品 B（雅詩蘭黛）：29 次，情緒 4.3
└── 結論：此 KOL 對你的品牌情緒最正面，且提及次數最多
```

```

  (品類契合度 × 0.3) +
  (歷史品牌提及正面度 × 0.25) +
  (受眾重疊度 × 0.2) +
  (互動率 × 0.15) +
  (性價比 × 0.1)

品類契合度：基於 KOL 過去提及的品牌品類分布
歷史品牌提及正面度：對類似品牌的平均情緒分數
受眾重疊度：KOL 粉絲輪廓與品牌目標 TA 的重疊程度
互動率：KOL 內容的平均互動率
性價比：預估效益 / 預估費用
```

## 3. 功能對比表

| 功能 | 現有 Cluster 系統 | 完整版 Ontology 系統 |
|------|-------------------|---------------------|
| **品牌搜尋** | 關鍵字搜尋，需手動輸入各種變體 | 自動合併所有別名（麥當勞=McDonald's=麥當當） |
| **產品層級分析** | ❌ 無法區分同品牌不同產品 | ✅ 自動識別產品層級（SK-II → 神仙水、青春露） |
| **Aspect 分析** | ❌ 不知道「服務好」是哪家店 | ✅ Aspect 精確關聯到特定 Entity |
| **KOL 品牌關係** | 只能看單篇貼文 | ✅ 聚合所有歷史提及，計算品牌契合度 |
| **競品比較** | 需手動搜尋、手動整理 | ✅ 一鍵生成多維度競品報告 |
| **智能 KOL 推薦** | 按粉絲數/互動率排序 | ✅ 基於 Entity Graph 的契合度推薦 |
| **即時警報** | Cluster 層級異常偵測 | ✅ Entity 層級精準警報 |
| **進階查詢** | SQL + 關鍵字 | ✅ OQL 語意查詢 |
| **趨勢發現** | 需人工觀察 | ✅ 自動識別上升中的品牌/話題 |

## 4. 商業價值

### 4.1 收費模式升級

| 現有模式 | Ontology 模式 |
|----------|---------------|
| 按 KOL 數據量收費 | 按品牌追蹤數 + 洞察深度收費 |
| 單價：NTD 5,000/月（基礎版） | 單價：NTD 30,000/月（品牌版） |
| 價值定位：數據庫工具 | 價值定位：品牌情報平台 |

### 4.2 客戶價值

| 客戶類型 | 現有價值 | Ontology 價值 |
|----------|----------|---------------|
| **品牌方** | 找 KOL 名單 | 完整品牌聲量追蹤 + 競品分析 + 即時警報 |
| **代理商** | KOL 數據查詢 | 智能 KOL 推薦 + 活動效果追蹤 |
| **KOL 自己** | 查看自己數據 | 品牌合作機會媒合 |

### 4.3 競爭壁壘

```
現有競爭壁壘：數據量（可被複製）

Ontology 競爭壁壘：
├── Entity Graph（需長期積累）
├── 語意理解能力（LLM + 領域知識）
├── 品牌別名庫（需持續維護）
└── KOL-Brand 關係網路（獨有數據資產）
```

## 附錄：User Story

### US-001: 品牌聲量追蹤
```
作為 品牌經理
我想要 看到我的品牌在社群上的完整聲量數據
以便 了解品牌在消費者心中的形象

驗收標準：
- 自動合併品牌所有別名的提及
- 顯示情緒分布（正面/中性/負面）
- 顯示 Aspect 級別的評價細節
- 顯示提及此品牌的 Top KOL
- 顯示時間趨勢圖
```

### US-002: 競品分析
```
作為 品牌經理
我想要 一鍵比較我的品牌與競品
以便 了解市場競爭態勢

驗收標準：
- 支援同時比較 2-5 個品牌
- 顯示各項指標對比（聲量/情緒/Aspect）
- 顯示 KOL 重疊分析
- 提供策略建議
```

### US-003: KOL 推薦
```
作為 行銷人員
我想要 系統推薦適合我品牌的 KOL
以便 提高合作成功率

驗收標準：
- 基於 KOL 歷史品牌提及進行推薦
- 顯示 KOL 與品牌的契合度分數
- 顯示 KOL 過去提過的類似品牌
- 提供預估合作效益
```

### US-004: 即時警報
```
作為 公關經理
我想要 在品牌出現負面輿情時即時收到通知
以便 及時應對危機

驗收標準：
- 設定警報觸發條件
- 顯示觸發原因和關鍵貼文
- 提供建議行動
- 支援多渠道通知（Email/Slack/App Push）
```
