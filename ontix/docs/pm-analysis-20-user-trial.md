## 1. 進展評估：Phase 7-8 的投入產出

### 分數變化：2.4 → 4.1（+1.7）是否合理？

**合理，但需要拆解來看。**

10 人模擬（Phase 0-6）的 2.4 分是在「連趨勢圖都沒有」的狀態下測的，那不是產品，那是 demo prototype。Phase 7-8 做的事情本質上是「讓產品從 demo 變成可用」，所以 +1.7 不是真正的進步，而是**還債**。

用更嚴格的框架來看：
- 2.4 → 4.1 只是從「完全不可用」到「勉強可看」
- 4.1 分代表 **0 個用戶會付費購買**（6 分以上才有付費意願）
- 20 人中只有 3 人給了 6 分，且都附帶「如果...就會到 7 分」的條件

### ROI 最高的改動

| 改動 | 開發估計 | 用戶感知 | ROI 判斷 |
|------|---------|---------|----------|
| **TrendChart 雙軸折線圖** | 前端 3-4 天 | 9/20 人好評，直接解決 10 人模擬的 #1 blocker | **極高** |
| **PeriodSelector (1W/4W/12W)** | 前端 1-2 天，後端 1 天 | 8/20 人好評，解鎖時間維度分析 | **極高** |
| **Aspect Drill-down → Mentions** | 前端 2-3 天，後端 1 天 | 16/20 人好評 Aspect 分析，鑽取是關鍵體驗 | **極高** |
| **Sparkline + Delta%** | 前端 2 天 | 11/20 人好評「所有數字都帶趨勢感」 | **高** |
| **What Changed 區域** | 前端 3-4 天 | 8/20 人好評，是 Pulse 的核心價值 | **高** |
| **Mention Sort + Sentiment Filter** | 前端 1-2 天 | 多數用戶自然使用，體驗流暢 | **高** |

### ROI 最低的改動

| 改動 | 開發估計 | 用戶感知 | ROI 判斷 |
|------|---------|---------|----------|
| **SOV Treemap** | 前端 3-5 天 | **0/20 人覺得有用**，U9 CMO 直接說「這不是我要的 SOV」 | **負 ROI，應砍掉或重做** |
| **Event Marker** | 前端 2-3 天，後端 1 天 | 只有 1/20 人注意到（U2 KOL AE） | **極低，藏太深** |
| **SentimentChart（獨立圖）** | 前端 2-3 天 | 與 TrendChart 功能重複，造成混淆 | **負 ROI，應合併或移除** |
| **Emerging 區域** | 前端 2 天 | U5 指出判斷邏輯錯誤（mention_count ≤ 8 ≠ 新品） | **低，邏輯需重寫** |
| **Topic Distribution (person)** | 前端 2 天 | 只有 KOL 本人（U7）會看，KOL 不是 ICP | **低，非目標用戶功能** |

### 小結

Phase 7-8 的高 ROI 改動集中在「基礎可視化」（趨勢圖、Sparkline、Period），這些本該在 MVP 就有。低 ROI 改動暴露出**產品判斷力問題**：在核心功能（匯出、比較）缺失的情況下，投資 SOV Treemap 和 Event Marker 是錯誤的資源分配。

## 2. 用戶分群與 ICP 重新定義

基於 20 人數據，識別出 4 個 segment + 2 個明確排除的 segment。

### Segment A：品牌方洞察消費者（Brand Insight Seekers）

| 維度 | 描述 |
|------|------|
| **代表用戶** | U5 電商PM(6分)、U10 新創品牌主(6分)、U12 CX主管(6分)、U19 商品開發(5分) |
| **核心 JTBD** | 「我要知道消費者對我的產品具體在意什麼、不滿什麼」 |
| **購買意願範圍** | 5-6 分，最接近付費 |
| **Ontix 匹配度** | **最高** — Aspect 分析是殺手功能，Watchlist + Alert 有價值 |
| **補最少功能就成交** | ① CSV/PDF 匯出 ② Aspect 時間趨勢 ③ 基本比較（2 entities） |
| **預估成交價** | 月費 NT$3,000-8,000（小品牌偏低，CX 主管偏高） |
| **成交機率** | 40-60%（補完上述 3 項後） |

**為什麼是第一優先**：這群人目前的替代方案是「手動看評價 + 焦點團體（每場 5-10 萬）」，Ontix 的 Aspect 分析直接取代了高成本的定性研究。他們不需要完美的報告，能匯出數據給團隊看就行。

### Segment B：代理商策略/提案人員（Agency Strategists）

| 維度 | 描述 |
|------|------|
| **代表用戶** | U1 行銷經理(5分)、U4 數據分析師(5分)、U8 策略企劃(5分)、U14 Planner(5分)、U13 業務總監(5分) |
| **核心 JTBD** | 「我要用數據做出有說服力的提案/報告，贏得客戶信任」 |
| **購買意願範圍** | 4-5 分 |
| **Ontix 匹配度** | 中等 — 分析深度有差異化，但「產出物」（報告、比較表）完全缺失 |
| **補最少功能就成交** | ① PDF/CSV 匯出 ② 比較功能 ③ API 文件 |
| **預估成交價** | 公司統一採購 NT$15,000-30,000/月（3-5 帳號） |
| **成交機率** | 25-35%（需要匯出 + 比較，且需要克服「已有 QSearch/i-Buzz」的慣性） |

**風險**：這群人的核心需求是「產出可交付的報告」，而非「深度分析」。他們會用 Ontix 的分析能力，但如果不能一鍵出報告，就會回到截圖 + PPT 的老路。QSearch/i-Buzz 的年約鎖定效應很強。

### Segment C：代理商/品牌方高管（Executive Decision Makers）

| 維度 | 描述 |
|------|------|
| **代表用戶** | U9 CMO(3分)、U16 PR Director(3分)、U20 代理商老闆(4分) |
| **核心 JTBD** | 「我要一個畫面說清楚品牌現況，帶進 board meeting / 年度策略」 |
| **購買意願範圍** | 3-4 分 |
| **Ontix 匹配度** | 低 — 需要比較、長週期趨勢、自訂 Dashboard，全部缺失 |
| **補最少功能就成交** | ① 比較功能 ② 年度時間範圍 ③ 匯出報告 ④ 多品牌分組 |
| **預估成交價** | NT$30,000-80,000/月（企業年約） |
| **成交機率** | 10-15%（需求太多，且已有 Brandwatch/i-Buzz 滿足基本需求） |

**判斷**：這是高客單價 segment，但 Ontix 目前離他們的需求太遠。不應作為 Q1-Q2 目標，但要在 Roadmap 中為他們鋪路。

### 明確不追的用戶

| 排除 Segment | 代表用戶 | 原因 |
|-------------|---------|------|
| **社群執行/危機監控** | U3 社群經理(3分)、U18 社群行銷專員(2分)、U6 公關Director(4分) | 核心需求是「原文連結 + 即時通知 + 去原文回覆」— 這是社群管理工具（Hootsuite/Sprout Social），不是輿情分析工具。Ontix 的 batch 架構從根本上不支持即時監控。**投入即時化的 ROI 極低。** |
| **KOL 本人** | U7 KOL(3分) | 需要的是 KOL analytics（Kolr/NoxInfluencer），不是品牌輿情。用戶自己說了：「不是為 KOL 設計的」。**完全不是 TAM。** |
| **微型品牌/個人** | U15 微型品牌主(2分) | 品牌太小不在系統中，沒有自助 onboarding，預算敏感。需要 self-serve + freemium，目前架構不支持。**CAC 會遠超 LTV。** |
| **媒體企劃** | U11 廣告公司媒體企劃(3分) | 需要受眾 demographics、平台聚合、日/小時粒度 — 全部是 Ontix 不擅長的維度。他們需要的是 Meta Business Suite + DV360。 |

## 3. Feature Priority 矩陣

### Impact 計算方式

Impact = 能讓多少用戶從「不買」（<6分）變「會買」（≥6分），加權考慮 segment 的 LTV。

### Effort 估計前提

- 1 個全職前端開發者 + 1 個全職後端開發者
- 1 天 = 1 人天

| Feature | Impact（幾人轉化） | Effort（前端+後端天數） | 類別 | 優先序 |
|---------|-------------------|----------------------|------|--------|
| **CSV/PDF 匯出** | 17/20 撞到，直接轉化 Seg A (4人) + Seg B 部分 (3人) = **~7人** | 前端 5d + 後端 3d = **8d** | **Quick Win** | **P0** |
| **比較功能（A vs B）** | 13/20 撞到，轉化 Seg B (3人) + Seg C 部分 (2人) = **~5人** | 前端 7d + 後端 5d = **12d** | **Strategic Bet** | **P1** |
| **原文連結（post URL）** | 6/20 撞到，但不影響 Seg A 的購買決策 = **~1-2人** | 前端 1d + 後端 2d（爬蟲存 URL）= **3d** | **Quick Win** | **P2** |
| **Aspect 時間趨勢** | 3/20 撞到，但含 U12 CX主管(6分→7分) = **~2人** | 前端 3d + 後端 2d = **5d** | **Quick Win** | **P2** |
| **交叉分析（topic×brand, aspect×brand）** | 4/20，含 U4 數據分析師 + U19 商品開發 = **~2人** | 前端 5d + 後端 4d = **9d** | **Strategic Bet** | **P3** |
| **API 文件** | 3/20，但 U17 Data Lead 是高影響決策者 = **~1人** | 後端 3d（Swagger/OpenAPI）= **3d** | **Quick Win** | **P3** |
| **修正 SOV Treemap** | 0/20 覺得有用，但修正成真正 brand SOV 可轉化 U9 CMO = **~1人** | 前端 3d + 後端 2d = **5d** | **Low Priority** | **P4** |
| **多帳號/權限** | 3/20，但 U20 代理商老闆是企業採購 gatekeeper = **~1人但高 LTV** | 前端 5d + 後端 8d = **13d** | **Strategic Bet** | **P4** |

## 4. 競爭定位分析

### Ontix 的真正差異化（用戶原話，不是我們自己說的）

從 20 人模擬中，用戶自發提到的獨特價值：

1. **「Aspect × Sentiment 分析是市場上最深的消費者洞察」** — 16/20 人好評
   - 競品做到「聲量 + 整體情緒」，Ontix 做到「哪個面向 × 正面/負面 × 具體怎麼說」
   - U12 CX主管：「Aspect 分析完美符合 CX 需求」
   - U14 Planner：「Aspect Negative = Unmet Needs，策略企劃的金礦」

2. **「Knowledge Graph 是 demo 殺手鐧」** — 12/20 人好評
   - U4：「市面上很少工具有這個」
   - U13 業務總監：「客戶都會 wow」
   - U20 代理商老闆：「這是跟客戶的 wow moment」

3. **「Ontology Rules 是真正的洞察，不是統計」** — 10/20 人好評
   - U6：「founder_risk + competitor_surge 是真正的洞察」
   - U4：「product_drags_brand 這種洞察競品沒有」

4. **「UI/UX 比所有台灣競品現代」** — 7/20 人好評
   - 多人對比 QSearch/i-Buzz 的介面，認為 Ontix 明顯更好

### 競爭優勢與劣勢

**vs QSearch**（代理商標配）
- 優勢：分析深度（Aspect級 vs 聲量級）、Knowledge Graph、UI
- 劣勢：沒有匯出、沒有比較、數據覆蓋率不透明、代理商生態綁定
- 用戶原話（U17 Data Lead）：「引擎比 QSearch 強，但缺透明度和數據可攜性。做為輔助工具 OK，取代不行。」
- **判斷**：短期不可能取代 QSearch，應定位為「深度洞察補充工具」

**vs i-Buzz**（品牌方標配）
- 優勢：自助分析 vs 月報制、Aspect 深度、即時性更好（雖然也是 batch）
- 劣勢：沒有報告、沒有專業分析師服務、品牌覆蓋面不確定
- 用戶原話（U14 Planner）：「Aspect + Topic 分析的深度比 i-Buzz 好」
- 用戶原話（U9 CMO）：「i-Buzz 至少有報告和競品比較」
- **判斷**：i-Buzz 的弱點是「月報制太慢、不能自助」，Ontix 正好打這個點

**vs Kolr**（KOL 工具）
- 不是直接競品，維度不同
- Ontix 有 KOL Attribution，但缺 engagement rate / reach / influence score
- **判斷**：不要試圖成為 KOL 工具，KOL Attribution 作為附加功能就好

### 建議定位語句

> **Ontix：消費者面向洞察引擎**
> 不只告訴你「多少人在討論」，而是告訴你「消費者在意什麼、不滿什麼、怎麼說的」。

更具體的 elevator pitch：

> 「QSearch 告訴你聲量漲了 20%，Ontix 告訴你是因為消費者抱怨保濕效果下降。i-Buzz 給你月報，Ontix 讓你自己隨時鑽下去看。」

不要用的定位：
- ~~「AI 驅動的輿情監控」~~ — 不是監控（batch 架構），且「AI 驅動」已經被用爛了
- ~~「一站式品牌管理」~~ — 做不到一站式，比較/匯出/多品牌全缺
- ~~「取代 QSearch/i-Buzz」~~ — 短期不可能，會引發直接比較然後輸

## 5. Go-to-Market 建議

### 第一批賣給誰

**Target：Segment A — 品牌方產品/CX/行銷主管**

具體畫像：
- 台灣中型品牌（年營收 1-10 億），有 5-15 人行銷/產品團隊
- 有至少 1 個人的工作包含「理解消費者對產品的評價」
- 目前用 i-Buzz 月報覺得太慢/太貴，或完全沒有輿情工具
- 行業：美妝、食品飲料、保養品、DTC 品牌

為什麼不先打代理商：
- 代理商需要匯出 + 比較 + 多品牌，全部都缺
- 代理商已有 QSearch 年約，switching cost 高
- 代理商用工具是為了給客戶看，但 Ontix 產出不了客戶 deliverable

### 定價建議

| Tier | 月費 | 包含 | 目標 |
|------|------|------|------|
| **Starter** | NT$5,000/月 | 5 entities 追蹤、1 帳號、Aspect 分析、Trend/Sparkline、基本匯出（CSV） | 新創品牌主、小型品牌 PM |
| **Pro** | NT$15,000/月 | 20 entities、3 帳號、PDF 報告、比較功能、API 存取 | 中型品牌行銷/CX 團隊 |
| **Agency** | NT$30,000/月 | 50 entities、10 帳號、多品牌分組、白標報告 | 代理商（Phase 9-10 後推出） |

定價邏輯：
- i-Buzz 月報 ~5 萬/月，Ontix Pro 定 1.5 萬是 i-Buzz 的 30%，但提供自助分析 + 更深的 Aspect 洞察
- QSearch ~3 萬/月，Ontix 不要正面對打價格，而是強調「不同維度的洞察」
- Starter tier 的存在是為了降低試用門檻，不是為了營收

### 用什麼 hook 進去

**Primary Hook：免費 Aspect 分析 Demo**

流程：
1. 跟品牌方說「給我你的品牌名，我 24 小時內給你一份消費者面向洞察報告」
2. 手動在系統中設定追蹤，跑 1-2 天
3. 截圖/錄影 Aspect Distribution + Negative Aspects + Ontology Insights
4. 展示會議：15 分鐘 demo，重點放在「你知道消費者最不滿的是什麼嗎？」
5. 提供 7 天免費試用

**為什麼這個 hook 有效**：
- U5（6分）：「Aspect 分析比我用過的任何工具都深」
- U10（6分）：「從來沒看過這種分析」
- U12（6分）：「Aspect 分析是市場上最好的」
- 這 3 位是最接近付費的用戶，他們被打動的都是 Aspect 分析

**Secondary Hook：Knowledge Graph 視覺衝擊**
- 在 demo 結尾放 Knowledge Graph，製造「wow moment」
- U13 業務總監：「客戶都會 wow」
- 但 Knowledge Graph 只是 wow factor，不是購買理由

### Sales Enablement 需求

| 項目 | 狀態 | 優先級 |
|------|------|--------|
| **Demo 錄影腳本**（3 分鐘，中文） | 缺 | P0 |
| **單頁品牌洞察報告 sample**（PDF） | 缺，需要匯出功能先完成 | P0 |
| **與 i-Buzz / QSearch 差異說明** | 缺 | P1 |
| **定價頁面** | 缺 | P1 |

| **API 文件** | 缺 | P2 |
| **Security / SLA 文件** | 缺 | P3（企業客戶需要） |
| **中文 Landing Page** | 缺 | P2 |

## 7. 風險與盲點

### 方法論限制

1. **這是模擬，不是真實試用**
   - 20 個「用戶」是 LLM 生成的 persona，不是真人
   - 真實用戶可能有完全不同的行為路徑（例如：可能根本不會發現某個功能）
   - 模擬傾向於系統化地測試每個功能，真實用戶可能 3 分鐘內就離開
   - **建議**：Q1 結束前必須有 5 個真人 pilot，用 session recording 觀察真實行為

2. **購買意願分數的校準問題**
   - 模擬的 6 分和真實的 6 分可能完全不同
   - 真實情境中，「比較功能」可能被「我已經有 QSearch 了」覆蓋掉
   - **建議**：不要把分數當絕對值，只看相對排序和趨勢

3. **10 人 → 20 人的增量信號**
   - 20 人模擬基本上驗證了 10 人模擬的結論（匯出/比較是 top blockers）
   - 新增的信號：Aspect 分析的價值被更多角色認可（16/20）
   - 但沒有出現「意外」的新 insight，可能代表模擬的多樣性不夠

### Survivorship Bias

1. **20 人全部是「會去試用輿情工具的人」**
   - 真實市場中，大量潛在用戶連輿情工具是什麼都不知道
   - 這 20 人已經是被篩選過的「有需求的人」
   - **盲點**：我們不知道「不知道自己需要輿情分析」的那群人長什麼樣

2. **美妝/消費品偏差**
   - 20 人幾乎全部在美妝、保養品、食品飲料行業
   - 不知道 B2B、SaaS、金融、醫療等行業是否有需求
   - **建議**：Phase 9 之前不要分心做其他行業

3. **台灣市場偏差**
   - 所有模擬都是台灣用戶、台灣競品
   - 定位和定價結論不能直接套用到其他市場

### 最危險的假設

按風險程度排序：

1. **「匯出 + 比較做完就能賣」（風險：極高）**
   - 這是模擬得出的結論，但真實世界中，功能 gap 只是不買的表面原因
   - 深層原因可能是：品牌根本不知道 Ontix 存在、不信任新創工具、不想換工具
   - **驗證方式**：3 月底前做完匯出，如果 pilot 轉化率 < 20%，就需要重新審視

2. **「Aspect 分析是真正的差異化」（風險：中高）**
   - 16/20 人好評，但好評 ≠ 付費
   - 如果 QSearch 或 i-Buzz 在 6 個月內加入類似功能，差異化消失
   - **驗證方式**：追蹤競品更新，確認 moat 是否可持續

3. **「數據覆蓋率夠用」（風險：中高）**
   - 4/20 用戶提到「不知道涵蓋多少平台」
   - U17 Data Lead 直接說「缺透明度」
   - 如果用戶發現自己品牌的數據不全，所有分析都失去意義
   - **驗證方式**：Pilot 階段主動告知覆蓋範圍，觀察用戶反應

4. **「品牌方願意為洞察付費」（風險：中）**
   - 台灣市場很多品牌方的輿情工具費用是代理商 bundle 的
   - 品牌方行銷部可能沒有獨立的工具預算
   - **驗證方式**：Pilot 談到定價時觀察反應

5. **「Batch 架構不是問題」（風險：低中）**
   - 目前的排除邏輯是「不做即時監控」，這是正確的
   - 但如果 Segment A 用起來後發現「昨天的負面今天才看到」是問題呢？
   - **驗證方式**：追蹤 pilot 用戶的使用頻率和滿意度

### 產品團隊的最大盲點

**我們一直在做「分析工具」，但用戶需要的是「工作流工具」。**

模擬中反覆出現的模式：
- 用戶能做分析 ✅ → 但不能帶走分析結果 ❌
- 用戶能看到問題 ✅ → 但不能採取行動 ❌
- 用戶覺得深度好 ✅ → 但不能融入現有工作流 ❌

如果繼續只做「更深的分析」而不解決「如何讓分析結果流入用戶的工作流」，4.1 分不會變成 7 分。匯出和比較不是「功能」，而是「讓分析有用武之地的管道」。
